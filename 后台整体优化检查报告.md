# 后台整体优化检查报告

生成时间：2026-05-06

## 本次已处理

- 统一后台主布局：侧栏、顶栏、内容容器、菜单选中态、移动端布局都改为更适合运营后台的紧凑排版。
- 统一基础视觉变量：主色、背景、边框、阴影、卡片、表格、表单控件改为更稳的生产后台风格。
- 优化公共组件：`DataTable` 默认横向滚动，避免列多时挤压；`FilterBar` 增加边框、阴影和移动端按钮/输入自适应。
- 清理后台无用依赖：移除未直接使用的 passport、jsonwebtoken、lodash、moment、qrcode、uuid、wechatpay-axios-plugin 等直接依赖和对应类型包。
- 补齐生产直接依赖：后台源码直接使用 `axios`、`express`、`socket.io`，已补为正式依赖，其中 `express` 固定到 `4.22.1`。
- 修复依赖清理后暴露的类型问题：`JwtGuard` 的 Request 类型已补充 `user` 字段。

- `202605060001_region_admin_enhance`：区域后台扩展字段，包括 `logo/address/latitude/longitude/radius/distanceLimit/studentOnly/regionType/isHot/regionCoverMode`、区域财务字段、页面布局 JSON 配置、私信/群聊/学生认证开关等。

- 生产基础日志：`ServerLog`、`SystemHealthSnapshot`、`ServerActionLog`、`UploadRecord`。
- 支付基础模型：`PaymentOrder`、`PaymentRefund`、`Reconciliation`、`PlatformLedger`、`MerchantSettlement`。

## 建议补的生产字段

这些不是当前构建阻塞项，但上线前建议排期：

- `AdminAccount`：建议增加 `deletedAt`、`loginFailCount`、`lockedUntil`、`passwordChangedAt`、`lastActiveAt`、`mfaEnabled/mfaSecret`，方便账号锁定、软删除、密码轮换和二次验证。
- `AdminLoginLog`：建议增加 `usernameSnapshot`、`regionId`、`deviceId`、`riskLevel`，账号不存在或被删除后也能追查登录行为。
- `PaymentOrder`：建议增加 `notifyStatus`、`notifyRaw`、`clientIp`、`subject`、`body`、`closedAt`、`closeReason`，便于支付回调幂等、对账和异常关单。
- `PaymentRefund`：建议增加 `notifyRaw`、`operatorId`、`reviewStatus`、`reviewedAt`，退款审核和第三方回调链路会更完整。
- `UploadRecord`：建议增加 `storageProvider`、`bucket`、`objectKey`、`etag`、`width`、`height`、`duration`、`deletedAt`，方便对象存储迁移、文件治理和内容审核。
- `ServerLog`：建议增加 `traceId`、`regionId`、`merchantId`、`requestBodyHash`，生产排查跨服务链路会更快，同时避免直接落敏感请求体。
- 核心业务大表建议补更多复合索引：如订单按 `regionId/status/createdAt`、支付按 `status/createdAt`、上传按 `hash`，避免后台列表在生产数据量上来后变慢。

## 生产注意项

- 本机命令行 Node 是 `18.20.5`，项目根 `package.json` 要求 `node >=20`；部署机和本地建议统一到 Node 20 LTS 或更高。
- `npm audit --omit=dev` 仍提示 Nest/COS/bcrypt 等链路存在漏洞，多数需要 Nest 10 -> 11、bcrypt 5 -> 6、COS SDK 方案升级，属于较大版本升级，建议单独做一轮兼容测试后处理。
- 当前目录不是 Git 仓库，无法用 Git diff 精确保护变更；建议后续把这个工作区纳入版本管理或从正式仓库拉取再改。
